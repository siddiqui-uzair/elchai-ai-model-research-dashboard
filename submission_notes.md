# Submission Notes

## Selected Model / Tool

Selected model: Qwen

I selected Qwen because it is an open source AI model family that may be useful for internal business workflows such as research support, document summarization, project coordination, coding support, and local or private AI assistant testing.

## Practical Output Created

I created a small Streamlit dashboard called **Elchai AI Model Research Dashboard**.

The dashboard compares Qwen, Kimi, MiniMax, and GLM based on possible business use cases, strengths, risks, and adoption recommendations. It also includes a use case selector that recommends which model may fit different Elchai business needs.

## Use Case for Elchai Group

The dashboard is designed to help Elchai Group evaluate which open source AI model could be tested first for internal workflows.

Possible use cases include internal research support, project coordination, document summarization, product support, and local or private AI assistant testing.

Departments that may benefit include Engineering, IT and Infrastructure, Product, Strategy, Operations, and Finance.

## Risks and Limitations

This project is only a beginner research dashboard and does not run the models directly.

Important risks include privacy concerns, security review needs, possible inaccurate AI outputs, integration difficulty, hardware cost for local models, and reliability issues.

Human review would be required before using AI generated outputs in real company workflows.

## Final Recommendation

My recommendation is to test Qwen first in a limited internal proof of concept using non confidential data.

It should not be used in production or with confidential company or client data until privacy, security, accuracy, integration, and reliability risks are reviewed.
